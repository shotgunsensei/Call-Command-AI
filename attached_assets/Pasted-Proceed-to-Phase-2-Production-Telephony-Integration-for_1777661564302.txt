PHASE 2A: TWILIO PRODUCTION INTEGRATION

Add environment variables:

TWILIO_ACCOUNT_SID=
TWILIO_AUTH_TOKEN=
TWILIO_API_KEY=
TWILIO_API_SECRET=
TWILIO_WEBHOOK_BASE_URL=
TWILIO_DEFAULT_RECORD_CALLS=true
TWILIO_VALIDATE_SIGNATURE=true

Add Twilio config validation utility:
- Detect whether Twilio is configured
- Never expose secrets client-side
- Show safe status in admin/settings UI

Create server endpoints:

POST /api/twilio/voice/incoming
Purpose:
- Receive incoming Twilio voice webhook
- Validate Twilio signature when enabled
- Identify channel by To phone number
- Create CallRecord immediately with:
  - channel_id
  - caller_phone from From
  - called_number from To
  - provider = twilio
  - provider_call_sid
  - status = incoming
  - call_direction = inbound
- Return TwiML response

POST /api/twilio/voice/status
Purpose:
- Receive call lifecycle updates
- Update CallRecord status:
  - ringing
  - in_progress
  - completed
  - failed
  - busy
  - no_answer
- Store duration and timestamps when available

POST /api/twilio/voice/recording
Purpose:
- Receive recording callback
- Store recording URL and metadata
- Trigger transcription + AI analysis pipeline
- Then execute assigned call flow automatically

POST /api/twilio/voice/transcription
Optional:
- Accept Twilio transcription callback if enabled
- Prefer internal transcription pipeline if OpenAI/Deepgram is configured

PHASE 2E: RECORDING INGESTION PIPELINE

When Twilio recording callback arrives:

1. Validate request
2. Find CallRecord by CallSid
3. Store:
   - recording_url
   - recording_sid
   - recording_duration
4. Set status = recording_ready
5. Download or stream recording server-side if required
6. Transcribe using configured transcription service
7. Analyze transcript
8. Execute assigned call flow
9. Create tickets/leads/tasks/actions based on flow
10. Mark status = ready/completed
11. Write flow logs and processing logs

Add retry-safe behavior:
- If same callback arrives twice, do not duplicate records
- If processing fails, store error message and allow manual retry

Add button on Call Detail:
- Retry processing
