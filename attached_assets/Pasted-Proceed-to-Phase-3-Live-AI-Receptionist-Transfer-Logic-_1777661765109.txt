PHASE 3C: TWILIO LIVE INTERACTION FLOW

Update POST /api/twilio/voice/incoming:

Instead of only recording/forwarding, support live receptionist behavior.

Channel behavior options:
- record_only
- forward_only
- voicemail_only
- ai_receptionist
- ai_screen_then_transfer
- ai_after_hours_intake

Add channel field:
- live_behavior

For live_behavior = ai_receptionist or ai_screen_then_transfer:

1. Create CallRecord
2. Create live_call_session
3. Return TwiML that:
   - Says greeting_script
   - Uses Gather speech input
   - Posts speech result to new endpoint:
     POST /api/twilio/voice/gather
   - Falls back to voicemail if no speech

Create endpoint:
POST /api/twilio/voice/gather

Responsibilities:
- Validate Twilio signature if enabled
- Find live_call_session by CallSid
- Append SpeechResult to transcript_live
- Run lightweight AI intake step
- Update collected_data
- Decide next action:
  - ask next question
  - transfer call
  - send to voicemail
  - create ticket/lead/task
  - escalate
  - end politely
- Return TwiML for next step

MVP can use Twilio Gather loops rather than full real-time streaming.
