Upgrade CallCommand AI from an audio processing app into a business automation platform.

Do NOT remove existing functionality. Expand it into a full workflow system.

CORE OBJECTIVE:
Transform call recordings into automated business actions (tickets, leads, tasks, notifications).

---

PHASE 1: AUTO INGESTION

Add ingestion sources:

1. Twilio integration (initial version can be simulated)
- Create endpoint: POST /api/ingest/twilio
- Accept recording URL + metadata
- Auto-create CallRecord

2. Email ingestion
- Create endpoint: POST /api/ingest/email
- Accept forwarded voicemail metadata
- Simulate parsing

3. Generic webhook ingestion
- POST /api/ingest/webhook
- Accept JSON payload with:
  - recording_url
  - caller_number
  - timestamp

All ingestion sources must:
- create CallRecord
- trigger transcription + AI analysis pipeline automatically

---

PHASE 2: DECISION ENGINE

Create a rules system:

Table: automation_rules
- id
- workspace_id
- name
- trigger_type (call_analyzed)
- conditions (JSON)
- actions (JSON)
- enabled

Example condition:
{
  "intent": "support",
  "priority": "high"
}

Example actions:
[
  { "type": "create_ticket" },
  { "type": "send_slack" }
]

Build rule evaluator:
- Runs after AI analysis
- Matches conditions
- Executes actions
